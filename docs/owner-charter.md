# SpiritHub Owner Charter

This document defines what the Owner team is trying to make true.

## Mission

Build SpiritHub.ro into a small, useful, profitable Romanian spiritual utility business.

The site should help visitors quickly answer personal-interest questions through calculators and
plain-language guides, then earn trust through speed, clarity, and repeat usefulness.

## Success Metric Hierarchy

Measure progress in this order:

1. Indexed pages and clean Search Console coverage.
2. Organic impressions for relevant Romanian queries.
3. Organic clicks and click-through rate.
4. Calculator usage and movement from guides into tools.
5. Return visits or repeated tool use.
6. AdSense eligibility and ad revenue.
7. One-time paid report potential after repeat use is proven.

Revenue matters, but the first bottleneck is trusted traffic.

## Decision Rules

- Traffic and trust come before monetization.
- Calculators and guides come before accounts.
- Small releases beat large speculative builds.
- Pages with search evidence beat pages based only on intuition.
- Ads must stay below the value moment and must not block results.
- Paid products wait until there is evidence of demand.
- Manual external setup should be reduced over time, but not by adding brittle automation too early.

## Owner Team Controls

The Owner team decides:

- business priorities,
- roadmap order,
- SEO/content direction,
- monetization experiments,
- manual tasks for the user,
- dev tasks for Codex CLI,
- what evidence is needed before the next decision.

The Owner team works mainly in Codex App.

## Dev Team Boundary

The Dev team decides how to implement a scoped task cleanly, but it should not invent a new business
direction. If the active task is unclear, it should update `docs/dev-brief.md` with the blocker
instead of building around the ambiguity.
