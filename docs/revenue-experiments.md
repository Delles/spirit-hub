# SpiritHub Revenue Experiments

Track monetization ideas here. Do not build an experiment until the Owner team marks it queued or
running.

## Status Values

- idea: possible future experiment.
- queued: chosen but not started.
- running: live and being measured.
- stopped: ended because it was not useful or not worth the complexity.
- won: successful enough to keep or expand.

## Experiment Template

```md
## Experiment name

- Status:
- Hypothesis:
- Success metric:
- Required user work:
- Required dev work:
- Risk:
- Review date:
- Outcome:
```

## Non-Personalized AdSense On Result Pages

- Status: idea
- Hypothesis: Once the site has traffic and AdSense approval, restrained ads below calculator results
  can create revenue without hurting trust.
- Success metric: Revenue per thousand sessions without a measurable drop in calculator completion.
- Required user work: Confirm AdSense approval and provide publisher/client ID plus slot IDs.
- Required dev work: Configure Vercel environment variables and verify layout stability.
- Risk: Ads too early may reduce trust or create GDPR complexity.
- Review date: After AdSense approval and meaningful traffic.
- Outcome: Pending.

## Paid Numerology Report

- Status: idea
- Hypothesis: Repeat users may pay for a richer personalized numerology PDF/report after using free
  calculators.
- Success metric: Click intent on a report CTA or survey-style waitlist conversion.
- Required user work: Decide acceptable payment/provider setup when evidence exists.
- Required dev work: Start with a no-payment interest CTA before building checkout.
- Risk: Building payments before traffic would waste time.
- Review date: After repeat-use evidence exists.
- Outcome: Pending.
